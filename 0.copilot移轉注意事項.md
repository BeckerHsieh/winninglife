# GitHub Copilot CLI 移轉注意事項

> 適用版本：GitHub Copilot CLI v1.0.60+  
> 更新日期：2026-06-07

---

## 一、移轉前準備（舊主機）

### 1. 確認所有設定位置
在舊主機執行以下 Copilot CLI 指令，確認現有設定：

```
/env      ← 顯示所有已載入的設定（指令檔、MCP、Skills、Agent 等）
/mcp      ← 查看 MCP 伺服器設定
/skills   ← 查看已安裝的 Skills
/agent    ← 查看可用的 Agents
/lsp      ← 查看 LSP 設定
```

---

## 二、需移轉的設定檔清單

| 項目 | Windows 路徑 | 說明 |
|------|-------------|------|
| 全域自訂指令 | `%USERPROFILE%\.copilot\copilot-instructions.md` | 全域 AI 行為指令 |
| MCP 伺服器設定 | `%APPDATA%\github-copilot\mcp.json` | MCP 工具設定 |
| Session 狀態資料夾 | `%USERPROFILE%\.copilot\session-state\` | 歷史 Session（選擇性） |
| 儲存庫層級指令 | `.github\copilot-instructions.md`（各 Repo 內） | 儲存庫專屬指令 |
| 其他自訂指令目錄 | 由環境變數 `COPILOT_CUSTOM_INSTRUCTIONS_DIRS` 指定 | 視情況移轉 |

> **注意**：`%USERPROFILE%` 通常為 `C:\Users\<使用者名稱>`  
> **注意**：`%APPDATA%` 通常為 `C:\Users\<使用者名稱>\AppData\Roaming`

---

## 三、移轉步驟

### Step 1：備份舊主機設定
```powershell
# 建立備份資料夾
New-Item -ItemType Directory -Path "D:\CopilotBackup"

# 備份全域指令
Copy-Item "$env:USERPROFILE\.copilot" "D:\CopilotBackup\.copilot" -Recurse

# 備份 MCP 設定
Copy-Item "$env:APPDATA\github-copilot" "D:\CopilotBackup\github-copilot" -Recurse
```

### Step 2：將備份複製至新主機
- 使用 USB、雲端硬碟或網路共享傳輸 `CopilotBackup` 資料夾

### Step 3：在新主機還原設定
```powershell
# 還原全域指令
Copy-Item "D:\CopilotBackup\.copilot" "$env:USERPROFILE\.copilot" -Recurse

# 還原 MCP 設定
Copy-Item "D:\CopilotBackup\github-copilot" "$env:APPDATA\github-copilot" -Recurse
```

### Step 4：重新登入 GitHub 帳號
```
/login
```

### Step 5：驗證設定是否正確載入
```
/env     ← 確認指令檔、MCP、Skills 都已載入
/mcp     ← 確認 MCP 工具可正常連線
```

---

## 四、MCP 伺服器移轉注意事項

- MCP 設定中若有**本機路徑**（如執行檔路徑、工作目錄），需依新主機路徑調整
- 若 MCP 工具需要額外安裝（如 `npm`、`uvx` 套件），需在新主機重新安裝
- 可用 `/mcp` 指令逐一測試各 MCP 工具是否正常運作

---

## 五、儲存庫相關設定

- `.github\copilot-instructions.md` 已在 Git 儲存庫內，Clone 後自動帶入
- `CLAUDE.md`、`GEMINI.md`、`AGENTS.md` 同樣隨儲存庫一起移轉

---

## 六、常見問題

| 問題 | 解法 |
|------|------|
| 移轉後登入失敗 | 重新執行 `/login` 完成 OAuth 授權 |
| MCP 工具無法連線 | 檢查 `mcp.json` 中的路徑是否正確 |
| 自訂指令未載入 | 確認 `copilot-instructions.md` 路徑正確，執行 `/instructions` 查看 |
| Skills 遺失 | 執行 `/skills` 重新安裝所需 Skills |
| Session 歷史消失 | Session 為本機資料，需手動複製 `session-state` 資料夾 |

---

## 七、更新 CLI 版本

移轉完成後建議將新主機 CLI 更新至最新版本：

```
/update
```
